The Problem

Family admin lives everywhere except where you need it.

Health insurance cards sit in drawers. A kid's Social Security number gets texted between spouses. Camp signups need a photo of the latest card. Passport details change, membership numbers go stale, and everyone quietly relies on the better-organized spouse to remember where the truth lives.

Text threads become storage.

Important details get buried in old messages, screenshots, and "can you send that again?" moments.

Shared docs decay fast.

A Google Doc feels practical for a week, then nobody knows whether the card photo or policy number is current.

Physical files are never nearby.

The one card or number you need is usually in a drawer, wallet, backpack, or email attachment somewhere else.

The Solution

A Siri-first vault for sensitive family info.

The key is not just storing the records. The key is asking naturally: "What's Mercer's health insurance?" or "What's my SSN?" As of WWDC 2026, Siri can reach App Intents and render custom snippet views, which makes a real answer card possible for the first time.

With on-device models and encrypted iCloud storage shared across authorized family members, sensitive data can stay obscured from the app maker, random third parties, and anyone who should not see it. Yes, Apple could eventually build this themselves. Until then, this is a damn fine solution.

The product promise is simple: the family source of truth, spoken back by Siri when that is enough, rendered as a card when a screen is available, and opened in FamFiles when the user needs the full record.

Implementation

Native where it matters, encrypted where it counts, intelligent at the edges.

FamFiles should feel like a small, trustworthy iOS app, not an enterprise records system. The stack is built around native Apple surfaces for the family-facing experience, a compact cloud layer for sync and coordination, and AI used only where it removes work from the user.

Front end: SwiftUI, App Intents, and system surfaces.

The primary client is a SwiftUI iPhone app with reusable record, family, person, and import views. The same core model powers widgets, Shortcuts, Share Sheet extensions, and Siri custom snippet views, so the app and the system answers stay visually and behaviorally aligned.

Back end: CloudKit first, service layer only where useful.

The default data plane is iCloud and CloudKit: private family databases, shared records, change history, conflict resolution, and offline-first sync. A small server can handle entitlement checks, analytics, support tooling, and non-sensitive orchestration, but the sensitive record payloads should not need to live on FamFiles-owned servers.

AI: extraction, matching, cleanup, and confidence.

AI reads incoming photos, PDFs, emails, text snippets, and screenshots; proposes record types; extracts fields; matches the right family member; detects whether an update supersedes older data; and asks a clarifying question when confidence is low. On-device Vision, Apple Foundation Models, and task-specific server models can be mixed depending on sensitivity and capability.

Siri: App Intents plus WWDC 2026 Apple Intelligence APIs.

FamFiles exposes people, records, documents, and household categories as App Entities with strong display representations. App Intents handle queries like "What's Mercer's health insurance?", return spoken dialog for voice-only contexts, and render custom SwiftUI snippet views when a screen is available. App and entity schemas make records more discoverable to Siri AI, while View Annotations can connect visible app screens to the right entities.

Ingestion: app capture plus Share Sheet everywhere.

Users can scan from inside FamFiles, but the more important path is sharing anything into the app: a card photo, camp email, Slack message, PDF, screenshot, or copied text. The Share Extension packages the item, runs local classification where possible, and routes it into the same review/update pipeline used by the main app.

Privacy: encrypt first, reveal narrowly.

Sensitive values should be encrypted before sync, scoped to authorized family members, and revealed only for the current task. Siri responses need a visual answer, a voice-only fallback, and a privacy mode that masks fields unless the user is authenticated and the context is appropriate. Audit trails show who added, viewed, copied, or changed a record.
